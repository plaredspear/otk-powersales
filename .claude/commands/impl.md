# 구현 모드

**IMPORTANT**: `.claude/guides/impl-mode.md`를 읽고 그 내용을 따라 진행합니다.

## 입력 파라미터 분석

사용자가 전달한 인자를 파싱합니다: $ARGUMENTS

- Part ID가 있으면 (예: `F59-M-P1`): 해당 Part의 구현을 시작
- Feature/플랫폼이 있으면 (예: `F59-알림-M`): 해당 스펙을 읽고 Part 확인 후 구현 시작
- 인자가 없으면: 사용자에게 어떤 작업을 구현할지 질문

## 워크플로우

1. **impl-mode.md** 가이드를 읽습니다
2. `/sync-main`으로 Git 동기화를 수행합니다
3. **스펙 선택적 로드** (impl-mode.md의 "스펙 선택적 로드 전략" 절차 수행):
   - 스펙 파일의 처음 ~50줄만 읽어 개요 + 승인 상태 확인
   - `## Part 분할 계획` 섹션을 Grep으로 찾아 읽고 현재 Part 범위 파악
   - 현재 Part에 해당하는 API 상세/화면 설계 섹션만 선택적으로 읽기
   - **스펙 파일 전체를 읽지 않습니다**
4. 컨텍스트에 맞는 플랫폼별 개발 가이드를 읽습니다
5. Part/Task 단위로 구현을 진행합니다
6. 각 Task 완료 시 테스트 실행 → 자동 커밋
7. 테스트 작성 시점에 `## 테스트 시나리오` 섹션을 지연 로드합니다
