# 릴리즈 노트

모바일 앱 버전별 릴리즈 노트를 보관한다. 각 파일은 `<X.Y.Z+N>.md` 형식이며
`mobile-v<X.Y.Z+N>` 태그와 1:1 대응한다.

## 작성 규칙

- **대상 독자**: 영업사원(앱 사용자). 개발/빌드 용어 금지.
- **포함 범위**: 직전 버전 이후 `feat`(새 기능) / `fix`(개선) 중 **사용자가 체감하는 변경만**.
  내부 변경(build/chore/refactor/style/test, 빌드·서명·flavor 등)은 제외.
- **문구**: 메뉴명·도메인 기준 사용자 친화 표현. "레거시 동등", "list.jsp" 등 내부 용어 제거.
- **섹션**: `## 새 기능`(feat) / `## 개선`(fix).

## 생성 방법

`/build-packages` 커맨드가 feat/fix 커밋을 AI 로 다듬어 이 폴더에 작성한다.
빌드 시 `mobile/scripts/build-packages.sh` 가 같은 버전의 파일이 있으면 자동 인식해
백엔드 앱 패키지 업로드의 `releaseNote` 로 사용한다.
